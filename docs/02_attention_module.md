# AI Task Breakdown: The Attention Module (`filter_attention.py`)

## Objective
Build `filter_attention.py`, the first node of the waterfall filter (Node 1). This module performs a **pure metadata filter** — zero video I/O — using the Charades-Ego CSV temporal action annotations to discard clips with insufficient object-manipulation activity. Only clips with meaningful action density survive to the expensive VLM stages.

## ⚠️ Important: What This Module Actually Measures
Charades-Ego's 157 action classes are **verb+object manipulation** activities (e.g., "Opening a door", "Holding a cup", "Putting clothes somewhere"). They are **not** social interaction labels. There are no "smiling", "laughing", or "hugging" classes in the dataset.

Our proxy heuristic: **activity density = attention**. If a clip is densely annotated with object-manipulation actions, the actor was actively performing tasks — and their body language during those tasks is what downstream modules analyze. Clips with sparse or no annotations are discarded as idle/ambiguous footage that yields no useful reward signal.

## Agent Instructions: Step-by-Step Tasks

### Task 1: Module Boilerplate & Data Loading
- **Action**: Create `filter_attention.py` with standard data science imports (e.g., `pandas`, `json`).
- **Action**: Write a function `load_manifest(filepath)` that reads `paired_clips_manifest.csv` (generated by `build_manifest.py`) into a Pandas DataFrame. The key columns are: `ego_video_id`, `tp_video_id`, `actions`, `length`.

### Task 2: Action Coverage Calculation
- **Action**: Parse the `actions` field for each clip. The field is a semicolon-separated string of `"class start end"` triplets (e.g., `"c008 11.90 21.20;c001 5.00 8.50"`).
- **Action**: Write a function `calculate_activity_coverage(actions_str, clip_length_s)` that:
  1. Parses each triplet into `(class_id, t_start, t_end)`.
  2. Computes the **union** of all annotated time intervals (handling overlaps correctly, e.g., via interval merging).
  3. Divides the total covered duration by `clip_length_s` to compute `activity_fraction`.
- **Note**: We intentionally use **all** 157 classes, not a subset. Every class represents meaningful physical activity that a robot could plausibly perform alongside a human. The filter's purpose is to reject *idle* clips, not to select specific action categories.


### Task 3: Primary Interaction Window Extraction
- **Action**: For each clip that passes the coverage threshold, calculate `primary_window` — the longest contiguous annotated action segment. Store `(t_start, t_end)` in the result. This window will be used by Module 03 (Engagement VLM) to know where to focus its analysis.
- **Action**: Write the core filter function `evaluate_attention(row, min_threshold=0.20)`. If `activity_fraction < 0.20` (i.e., less than 20% of the clip contains any annotated action), flag the clip as `ATTENTION_FALSE`.
- **Rationale**: The 20% threshold is deliberately low — Charades annotations are temporally sparse compared to continuous gaze tracking. Even a 30-second clip with 6 seconds of annotated activity provides usable signal.

### Task 4: Output & Pipeline Interfacing
- **Action**: Write the evaluation results to an output file `attention_results.json` as a JSON array:
  ```json
  [
    {
      "ego_video_id": "ABCDEEGO",
      "tp_video_id": "ABCDE",
      "passed_attention": true,
      "activity_fraction": 0.45,
      "primary_window": {"t_start": 5.0, "t_end": 21.2}
    }
  ]
  ```
- **Success Criteria**: The script processes the entire manifest using only CSV metadata (no video I/O) and completes within seconds. Clips with insufficient activity coverage are discarded before any GPU/VLM work begins.
