# AI Task Breakdown: The Engagement Module (`analyze_engagement.py`)

## Objective
Develop the logic to extract the actor's observable "Cognitive State" (`Focused`, `Neutral`, `Startled`) from filtered clips using a Temporal Vision-Language Model. Only clips that passed Node 1's attention filter are processed here.

## ⚠️ Critical: Use the Third-Person Paired Video
The Charades-Ego egocentric clip shows the world **from the actor's own eyes** — you **cannot** see the actor's face, shoulders, or torso in this view. The engagement module **must** load the **third-person paired video** (via `tp_video_id` / `tp_video_path` from `paired_clips_manifest.csv`) where the actor's full body and facial expressions are visible to the camera.

## Agent Instructions: Step-by-Step Tasks

### Task 1: VLM Integration Setup
- **Action**: Create a new Python file `analyze_engagement.py`.
- **Action**: Load `attention_results.json` (generated by `filter_attention.py`). Filter to only entries where `passed_attention == true`.
- **Action**: For each passing entry, look up `tp_video_path` from `paired_clips_manifest.csv` using `tp_video_id`. This is the video file you will feed to the VLM.
- **Action**: Integrate **Ollama via `ollama.chat()`** for cognitive state classification. Use the `moondream` model for efficient inference.
- **Action**: Sample frames from the **third-person video** within the `primary_window`. Seek using `cap.set(cv2.CAP_PROP_POS_MSEC, t_start * 1000)`. Sample frames at 1 FPS (max 5 frames) to respect VLM context limits.
- **Action**: Implement a function `generate_engagement_prompt()` that returns a strict-JSON formatting prompt.
- **Action**: Execute inference using `ollama.chat()` to ensure correct handling of base64 image strings in the message payload.


### Task 3: Output Parsing
- **Action**: Write a robust parser `parse_vlm_response(response_text)` that parses the JSON output to strictly classify into `FOCUSED`, `NEUTRAL`, or `STARTLED`. 
- **Action**: Implement error handling: Catch `JSONDecodeError` or unmapped responses (VLM hallucinations) and assign them to an `UNKNOWN` state. Log all `UNKNOWN` results for manual review.

### Task 4: Data Formatting
- **Action**: Save the output to `engagement_results.json` as a JSON array:
  ```json
  [
    {
      "ego_video_id": "ABCDEEGO",
      "tp_video_id": "ABCDE",
      "state": "FOCUSED",
      "t_start": 5.0,
      "t_end": 21.2,
      "confidence": 0.85
    }
  ]
  ```
- **Action**: The `confidence` field should be derived from the VLM's output logprobs if available, otherwise default to `1.0` for successful parses and `0.3` for `UNKNOWN` states.
- **Success Criteria**: The VLM categorizes a batch of clips into the three designated cognitive states using the third-person view, without causing Mac OOM (Out of Memory) crashes or swapping freezes.

## ✅ Resolved Pipeline Issues (Audit 2026-04-17)

| Issue | Root Cause | Resolution |
|-------|------------|------------|
| **VLM API Bug** | `ollama.generate()` failed with base64 images. | Switched to `ollama.chat()` with message payloads. |
| **JSON Payload** | `format='json'` was version-dependent in `generate`. | Verified support in `ollama.chat()`. |
| **Hard-Coded Limit** | `LIMIT = 5` testing artifact throttled production. | Added `argparse` `--limit` flag (default=None). |
| **Library Conflicts** | `mlx-vlm` and `transformers` v4.49+ clashed. | Pivoted to Ollama/Moondream backend. |
